## 0. 이게 뭔데

"AI가 혼자서 도서관에 갇혀서 100년 동안 공부하고 나오면 뭐가 될까?"

이 미친 질문에 대한 답을 찾기 위한 실험이다. 기존 AI 학습은:
- 지도학습: 인간이 정답 달아줌 → 인간 수준에 갇힘
- RLHF: 인간 피드백 → 인간의 편견 학습  
- 증류: 큰 모델에서 작은 모델로 → 원본보다 나을 수 없음

AGISTI는 **선생님이 없다**. 모델이:
1. 스스로 문제를 만들고
2. 스스로 풀고
3. 스스로 채점하고
4. 스스로 가중치를 수정한다

이번 Phase 2 실험에서 **720억 개 파라미터짜리 모델**이 실제로 이걸 해냈다.

### 2.2 활성화된 천장 돌파 시스템

Phase 1에서는 기본 루프만 돌았지만, Phase 2에서는 디자인 문서의 4단계 천장 돌파 시스템을 활성화:

| 단계 | 이름 | 상태 | 설명 |
|------|------|------|------|
| Level 1 | 외부 문제 소스 | ✅ 활성 | GSM8K, ARC, MMLU 데이터셋에서 검증된 문제 주입 |
| Level 2 | RAG 수술 | ✅ 활성 | 실패 문제 기반 문서 검색 → 컨텍스트 대비 수술 |
| Level 3 | 교차 수분 | ❌ 비활성 | 참조 모델 미지정 (다음 Phase에서 활성화 예정) |
| Level 4 | 복합 기술 발견 | ✅ 활성 | 개별 스킬은 OK인데 합치면 실패하는 문제 탐색 |

### 2.3 핵심 안전 장치

- **수술 예산** (β = 0.01): 한 번에 가중치를 너무 많이 못 바꿈
- **QuickBench 관문**: 수술 후 벤치마크 검증 실패 시 즉시 롤백
- **재앙 감지기**: 점수가 급격히 떨어지면 비상 복구
- **Frozen Zone**: 건드리면 안 되는 핵심 레이어 보호 (이번엔 시간 절약을 위해 건너뜀)

## 3. 돌파 과정 (삽질 연대기)

### 3.1 시도 1: bfloat16 풀 로딩

| 항목 | 결과 |
|------|------|
| 모델 적재 | ✅ 148초만에 3장 분배 완료 |
| GPU 사용량 | 47 + 48 + 47 = 142GB |
| Iteration 0 수술 | ✅ 수락!!! (QuickBench 42%) |
| Iteration 1 | ❌ OOM 사망 (462MB 할당 실패, 151MB 남음) |

**교훈**: 72B bfloat16은 GPU당 48GB를 먹어서 수술 시 VRAM이 0에 수렴한다.

### 3.2 시도 2: bfloat16 + 메모리 최적화

- 문제 수 축소, 벤치 문제 감소, 가상 훈련 스텝 줄임
- 결과: **또 OOM**. 33MB 차이로 터짐.
- 72B bfloat16은 근본적으로 3장에 수술하기엔 빡빡함.

### 3.3 시도 3: 4비트 양자화

- `BitsAndBytesConfig(load_in_4bit=True)` 적용
- 결과: `'Qwen2ForCausalLM' object has no attribute 'set_submodule'`
- 파이토치 2.4.1에 `set_submodule` 메서드가 없음!

### 3.4 시도 4: 8비트 양자화

- `BitsAndBytesConfig(load_in_8bit=True)` 적용
- 같은 `set_submodule` 에러!
- **해결**: monkey-patch로 `torch.nn.Module`에 `set_submodule` 직접 구현!

```python
# run_phase2.py 상단에 추가한 monkey-patch
import torch.nn as nn
if not hasattr(nn.Module, 'set_submodule'):
    def _set_submodule(self, target, module):
        atoms = target.split('.')
        mod = self
        for item in atoms[:-1]:
            mod = getattr(mod, item)
        setattr(mod, atoms[-1], module)
    nn.Module.set_submodule = _set_submodule
```

### 3.5 시도 5: 8비트 + monkey-patch ← 🎉 **성공!!!**

드디어. 드디어!!! 72B가 8비트로 깨끗하게 올라가고, 수술도 성공하고, OOM도 안 나고, 벤치마크도 통과했다.

## 5. bfloat16 시도에서의 추가 증거

8비트 이전에 bfloat16으로 돌린 결과도 중요한 증거다:

### 5.1 Iteration 0 (bfloat16)

- 프로브: **48.0%**
- QuickBench: **42.0%** → 수술 수락!
- 6개 레이어 수정, delta norm 0.0041

### 5.2 Iteration 1 (bfloat16, OOM 전)

- 프로브: **49.2%** ← **48.0%에서 상승!!!**
- 실패 수: 13개 → **12개로 감소!!!**

> **🔥 핵심 증거: 수술 후 프로브 점수가 48.0% → 49.2%로 상승했다!!!**  
> 72B 모델이 자기 수술을 통해 **실제로 더 똑똑해졌다.**  
> 이것은 선생님 없는 자기 개선이 가능하다는 직접적 증거다.
